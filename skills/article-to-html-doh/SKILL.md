# article-to-html-doh

Render a document into a **self-contained paper proposal HTML file** and publish it to the DOH blog server.

## Hard Trigger Rule

Use this skill **only** when the user's request explicitly contains:

- `/doh`
- standalone `doh`

If the user asks for HTML/article rendering without `/doh` or standalone `doh`, do **not** use this skill.

## Flow

1. **Check config.** Read `.env` in this skill directory. If any required value is missing, ask the user to provide it before publishing:
   - `DOH_HOST`
   - `DOH_USER`
   - `DOH_PASSWORD` or working SSH key access
   - `DOH_PUBLIC_BASE_URL`
   - `DOH_REMOTE_DIR`
   - `DOH_BASE_PATH`
2. **Get the source.** If the user gave a file path, read it. If the draft is in conversation context, use that.
3. **Extract skeleton.** Title, subtitle, TL;DR, sections, date, category, whether figures/tables/interactive behavior are needed.
4. **Read the template.** Use `references/template.html` as the scaffold.
5. **Read component references as needed.**
   - `references/components.md` for callouts, cards, tables, footers.
   - `references/svg-figures.md` if inline SVG figures are needed.
   - `references/interactive.md` only if the article clearly benefits from interaction.
6. **Generate one complete HTML file.**
   - Keep all CSS/SVG/JS inline.
   - Include `<title>`, `<meta name="description">`, and one clear `<h1>`.
   - Write locally first, using a safe filename.
7. **Publish.** Run:

   ```bash
   node <skill-dir>/scripts/publish-html.mjs <html-file> --category <category>
   ```

   The script uploads to:

   ```text
   DOH_REMOTE_DIR/<category>/<YYYY-MM-DD-slug>.html
   ```

8. **Report.** Return the local HTML path and the public URL.

## Design Invariants

- **Single file.** All CSS / SVG / JS inline. No external fonts, CDNs, or remote images unless the user explicitly requires them.
- **Paper palette.** Keep `--paper: #f7f7f5` and `--ink: #1a1a1a`.
- **Serif body + monospace meta.** Body uses `ui-serif`; metadata slots use `ui-monospace`.
- **Numbered sections.** Use `<h2><span class="num">01</span>Section title</h2>`.
- **Figures need captions.** Use `<figcaption><span class="fig-num">FIG N</span>caption</figcaption>`.
- **TL;DR near top.** If the source has no TL;DR, condense the first one or two paragraphs.
- **No AI watermark** unless requested.

## Naming

- Category: use the user's category if provided; otherwise infer from the topic, defaulting to `notes`.
- Filename: generated by the publish script from the HTML title and date.
- Date: use the explicit article date if present; otherwise use current local date.

## Common Mistakes

- Do not use this skill without `/doh` or standalone `doh`.
- Do not publish if `.env` is missing required server config.
- Do not upload a static homepage; the blog index is generated by the server.
- Do not put server passwords into generated HTML.

## File Manifest

- `.env` — local private server config, not intended for git.
- `.env.example` — non-secret config template.
- `scripts/publish-html.mjs` — uploads one generated HTML file.
- `references/template.html` — full CSS + document scaffold.
- `references/components.md` — callout/card/table/footer snippets.
- `references/svg-figures.md` — SVG figure skeletons.
- `references/interactive.md` — optional JS snippets.
- `assets/example.html` — completed reference render.
