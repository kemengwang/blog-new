01问题

js是脚本语言 可以在浏览器这个环境里直接执行 但是直接写js文件 会有各种各样的不便性

源码和第三方library在html文件中用script标签引入，这种做法随着项目规模的扩大会带来很多问题，主要问题为:

  • 变量都在全局作用域，容易命名冲突
  • 需要处理好文件之间的依赖，代码引入要注意加载顺序
  • http请求过多，出现性能问题
  • 代码维护复杂

为此打包必不可少 有点像编译环节

好处

  1. 减少网络请求
  2. 优化加载性能
  3. 模块化开发
  4. 支持新特性
  5. 代码分割和懒加载
  6. 使用现代化框架
  7. 使用静态类型语言
  8. 对各类浏览器做兼容
  9. 便利使用各类流行库(前端生态)
  10. 制定各种自定义规则

04主流库使用打包工具分析

React

作为前端最流行的框架 ，他采用的是rollup作为整体库的构建

可见核心逻辑在 buildEverything中

核心就是对需要构建的bundles列表 执行createBundle 创建所有需要的bundle

createBundle：可以看到利用rollup.watch 和 rollup.rollup rollup.write 相关js api完成bundle构建

material-ui

纯采用babel作为打包构建 打包后保留了原始的项目结构

好处 本身就是组件的形式组成项目 打包后 可以为每个组件添加package.json 方便用户直接引入子组件

vue3

rollup yyds

element-plus

整体采用gulp组织流程

内部任务结合rollup js api

dev模式采用的 vite

next.js

由webpack打包 全面转向 turbo

turbo由rust编写的 增量打包库

ant-design-mobile

使用gulp 生态 完成cjs es 打包 使用vite、webpack 相关js api 打包bundle umd

05脑洞大开系列

electron

electron是一个客户端跨平台解决方案(c编写)

目前qq、bilibili pc端 均采用该技术

gn: Generate Ninja 用于生成 Ninja 的构建文件. Ninja 是一个小型的构建系统.

可以看倒electron在整体构建中 其中一部分有对前端代码的打包，采用的webpack打包 webpack.gni其内容就是执行了npm命令 执行webpack打包

react-native

C++编写

它允许开发者使用 JavaScript 和 React（一种用于构建用户界面的 JavaScript 库）编写应用，然后将这些应用编译成原生应用，可以在 Android 和 iOS 设备上运行 (支持热重载)

主要使用了babel来转化库里前端相关的代码

uni-app

taro
