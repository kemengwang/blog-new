总结业务中曾用到的一些性能优化手段

01加速优化方案

  • 使用ssr重构
  • app内使用离线包或者网页提前dns解析、tcp建连，单页面、静态资源使用cdn加速。
  • gzip 压缩(静态文件压缩超70%),压缩后可以直接用于网路传输
  • 动态引入chunk,首屏不需要的内容，都可以动态引入，减少首屏js大小
  • 第三方库按需引入，vender太大导致变慢，可以看哪些第三方太大，单独去引入
  • 图片使用avif、webp，图片懒加载 ，也可以打包降低图片质量，使用plugin，不推荐，打包时将图片传到cdn节点上，加速访问
  • 小图标使用矢量图，本身就比较小，直接使用svg矢量图，减少网络请求，如果没有打包时base64小图标也可以
  • Web Worker: 开另一个线程优化大计算 阻塞渲染的东西
  • 防抖 & 截流
  • 压缩 JS 和 CSS，vite打包可以直接开启
  • css策略: CSS 选择器是从右向左进行匹配，最佳使用类选择器，不要嵌套超过3层。像是通配符、标签选择这些其实都慢。动画使用css3动画。打包时css做好合并。transform、opacity、will-change合成层面减少卡顿的现象
  • 请求优化 使用缓存、并发请求等等
  • script执行优化，添加async defer 或者type='module'这些，这里vite框架做了，并且还右降级代码使用plugin-legacy
  • 重要的数据preload 通过link
  • 打包自带的摇树

02react加速

  • 使用hook集成业务逻辑，做到逻辑复用
  • Memo 和 useCallback
  • Fragment
  • key
  • 组件颗粒度
  • 错误边界
  • React.lazy和React.Suspense
  • startTransition、useTransition、useDeferredValue
