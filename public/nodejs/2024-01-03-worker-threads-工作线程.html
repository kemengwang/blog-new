01Worker threads

02Class

MessagePort

消息端口 代表双向通信频道的一端

Event

close : 任何一方断开连接 就会触发'close'事件

const { MessageChannel } = require('node:worker_threads');
const { port1, port2 } = new MessageChannel();

port2.on('message', (message) => console.log(message));
port2.on('close', () => console.log('closed!'));

port1.postMessage('foobar');
port1.close();

// Prints:
//   foobar
//   closed!

message: postMessage(value) 会派发 message事件 value会被克隆传入

messageerror : postMessage(value) value值反序列化 无法解析 会被触发

Method

port.close() 关闭端口

port.postMessage(value[, transferList])

  • value :any （克隆目标（序列化形式 克隆））
  • transferList : Object[]. （移动 使用此法 发送端不可访问 所有权转移 注意如果要使用value里要有）
const assert = require('node:assert');
const {
  Worker, MessageChannel, MessagePort, isMainThread, parentPort,
} = require('node:worker_threads');
if (isMainThread) {
  const worker = new Worker(__filename);
  const subChannel = new MessageChannel();
  worker.postMessage({ hereIsYourPort: subChannel.port1 }, [subChannel.port1]);
  subChannel.port2.on('message', (value) => {
    console.log('received:', value);
  });
} else {
  parentPort.once('message', (value) => {
    assert(value.hereIsYourPort instanceof MessagePort);
    value.hereIsYourPort.postMessage('the worker is sending this');
    value.hereIsYourPort.close();
  });
}

port.ref() port.unref() port.hasRef() 用于保持端口为活动状态 防止程序退出 port.start() 设置message监听时自动调用

MessageChannel

异步的双向通信通道

const { MessageChannel } = require('node:worker_threads');

const { port1, port2 } = new MessageChannel();
port1.on('message', (message) => console.log('received', message));
port2.postMessage({ foo: 'bar' });
// Prints: received { foo: 'bar' } from the `port1.on('message')` listener

有port1, port2 两个端口(messagePort) 然后双向通信

BroadcastChannel

异步的一对多通信

new BroadcastChannel(name)

连接的通道名字

method

broadcastChannel.close() 关闭 broadcastChannel.onmessage 消息监听 broadcastChannel.onmessageerro broadcastChannel.postMessage(message) 消息派发 broadcastChannel.ref() broadcastChannel.unref()

'use strict';

const {
  isMainThread,
  BroadcastChannel,
  Worker,
} = require('node:worker_threads');

const bc = new BroadcastChannel('hello');

if (isMainThread) {
  let c = 0;
  bc.onmessage = (event) => {
    console.log(event.data);
    if (++c === 10) bc.close();
  };
  for (let n = 0; n < 10; n++)
    new Worker(__filename);
} else {
  bc.postMessage('hello from every worker');
  bc.close();
}

主线程、子线程 都定义了hello的通道

Worker

new Worker(filename[, options])

filename: string | url 主要传路径 绝对/相对都可以 options: object

  • workerData: any 传递数据 序列化克隆进线程中 require('node:worker_threads').workerData. 线程中可以直接引入
  • argv: any[] 追加在 process.argv
  • eval: boolean 开启时 filename变为可执行string块
  • transferList : 同上 转移必要的所有权 如果workerData里有端口 一定要传
  • stdin: boolean 为true worker.stdin提供可写流 Worker 内部作为 process.stdin出现
  • stdout: boolean 为 true worker.stdout 不会自动传输到父进程的 process.stdout
  • stderr: boolean 为 true worker.stderr 不会自动传输到父进程的 `process.stderr
  • resourceLimits 设定物理资源限制
  • 还有其他属性: link

Event

error: 线程中抛出异常时 触发 并且线程中断

  • err Error

exit: 线程中调用 process.exit() 线程中断

  • exitCode integer 退出为1

message、messageerror: 同上文

online: 线程 开始执行js代码时触发

method

  • worker.getHeapSnapshot([options]) 获取堆快照信息
  • worker.performance 一个对象查询线程实例 性能数据
  • performance.eventLoopUtilization([utilization1[, utilization2]]) 用于获取的事件循环利用率
  • worker.postMessage(value[, transferList])
  • worker.ref() unref() 标记关键\ 非关键 （是否需要等线程完成 程序才退出）
  • worker.resourceLimits 一个对象 资源占用内容
  • worker.stderr worker.stdin worker.stdout
  • worker.terminate() 快速终止线程 返回promise resolve的是退出码 在exit事件派发时激发promise

const {
  Worker,
  isMainThread,
  setEnvironmentData,
  getEnvironmentData,

if (isMainThread) {

} else {

}

getEnvironmentData获取设置的值 在线程中 (注意 设置的值只有map的key可以设置的值) isMainThread 是否是主线程

receiveMessageOnPort(port)

接受一条消息 从 消息端口中

注意 消息派发如果没有监听 那么消息会堆积在消息队列中 receiveMessageOnPort会获取最老的一条消息

const { MessageChannel, receiveMessageOnPort } = require('node:worker_threads');
const { port1, port2 } = new MessageChannel();
port1.postMessage({ hello: 'world' });

console.log(receiveMessageOnPort(port2));
// Prints: { message: { hello: 'world' } }
console.log(receiveMessageOnPort(port2));
// Prints: undefined

SHARE_ENV

const { Worker, SHARE_ENV } = require('node:worker_threads');
new Worker('process.env.SET_IN_WORKER = "foo"', { eval: true, env: SHARE_ENV })
  .on('exit', () => {
    console.log(process.env.SET_IN_WORKER);  // Prints 'foo'.
  });
