02二、 泛型和特征

A、泛型

根据定义 传参 来约定声明的<T> 是啥类型

编译时的产物 无性能消耗

1.各种场景

函数使用泛型 函数名后进行声明
fn add<T>(a:T, b:T) -> T { a }

未确定类型不能相加 （除非限定特征）

结构体中使用
struct Point<T> {
    x: T,
    y: T,
}

fn main() {
    let integer = Point { x: 5, y: 10 };
    let float = Point { x: 1.0, y: 4.0 };
}
枚举中
enum Option<T> {
    Some(T),
    None,
}
方法中

impl<T> 声明T Point<T> 实际就是完整的类型 前面声明T传给了它 还能给方法函数用 <V, W> 和T不冲突

struct Point<T> {
    x: T,
    y: T,
}

impl<T> Point<T> {
    fn x<V, W>(&self) -> &T {
        &self.x
    }
}

fn main() {
    let p = Point { x: 5, y: 10 };

    println!("p.x = {}", p.x());
}

B、特征

一种行为的抽象

特征定义了一组可以被共享的行为，只要实现了特征，你就能使用这组行为

1.定义

不同类型 具有相同行为 为此我们可以定义一个特征，为这些类型实现这些行为

定义特征是把一些方法组合在一起，目的是定义一个实现某些目标所必需的行为的集合。

trait Summary {
    fn summarize(&self) -> String;
}

关键字：trait

特征可以定义行为看起来的样子 也可以提供默认实现

2.实现

实现

impl Summary for Post 为Post实现Summary特征

pub trait Summary {
    fn summarize(&self) -> String;

pub struct Post {

    pub content: String, // 内容

impl Summary for Post {
    fn summarize(&self) -> String {
        format!("文章{}, 作者是{}", self.title, self.author)

特征组合

实现特征中某个方法 便可使用其他所有方法

pub trait Summary {
    fn summarize_author(&self) -> String;

    fn summarize(&self) -> String {
        format!("(Read more from {}...)", self.summarize_author())
    }
}


impl Summary for Weibo {
    fn summarize_author(&self) -> String {
        format!("@{}", self.username)
    }

3.孤儿原则

为某个类型实现某个特征 那么类型 或者 特征 至少有一个是定义在当前作用域

4.特征约束

pub fn notify<T: Summary>(item: &T) {
    println!("Breaking news! {}", item.summarize());
}

// 多重约束
pub fn notify<T: Summary + Display>(item: &T) {
    println!("Breaking news! {}", item.summarize());
}

对泛型进行约束 传入的类型必须具有该特征

可以使用语法糖

pub fn notify(item: &impl Summary) {
    println!("Breaking news! {}", item.summarize());
}

pub fn notify(item: &(impl Summary + Display)) {}

    }

5. 调用特征方法需要引入特征

最常用的标准库中的特征通过 std::prelude 模块提前引入到了当前作用域

use std::convert::TryInto; //可以删除 在prelude里

fn main() {
  let a: i32 = 10;
  let b: u16 = 100;

  let b_ = b.try_into()
            .unwrap();

  if a < b_ {
    println!("Ten is less than one hundred.");
  }
}

6. 特征对象

类型定义：Box<dyn Draw> / &dyn Draw

依赖隐式转换 (发生条件Deref有讲):

常规隐式转换& 特征对象特殊box包裹也可 draw1传入Box<T> 转换为Box<dyn Draw> draw2传入 &T 转换为 &dyn Draw (假设T的特征为Draw)

...

fn draw1(x: Box<dyn Draw>) {
    x.draw();
}

fn draw2(x: &dyn Draw) {
    x.draw();
}

fn main() {
    let x = 1.1f64;
    let y = 8u8;

    draw1(Box::new(x));
    draw1(Box::new(y));
    draw2(&x);
    draw2(&y);
}

&&a -> &a

&Box(String(a)) -> &String(a) -> &a

常见隐式转换发生在方法调用 函数调用 定义变量时指定借用类型 赋予时可以发生
